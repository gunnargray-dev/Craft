# Craft

A SwiftUI iOS design library app that demonstrates high-craft UI principles. Every screen validates a specific design skill — motion, components, typography, color, spacing, navigation, interactions, accessibility, and platform adaptation.

## The App

4 tabs, 29 views, ~4,800 lines of SwiftUI:

- **Components** — Button gallery (4 levels × 4 sizes), text inputs (5 states), cards with skeletons, lists with swipe actions, sheets with detents, empty states
- **Foundation** — Type scale (11 iOS styles), type hierarchy demo, semantic color palette with contrast ratios, spacing ruler (4pt grid), interactive border radius
- **Motion** — Spring playground (5 presets with sliders), enter/exit asymmetry, staggered reveals, shared element transitions, haptic feedback, gesture demos
- **Audit** — Dynamic Type preview (3 sizes), contrast checker, live size class viewer, interactive accessibility checklist

### Requirements

- Xcode 16+
- iOS 18.0+
- No external dependencies

### Run It

```
open Craft.xcodeproj
```

Build and run on simulator or device.

## The Skills

The `skills/` folder contains the design skills that produced this app. They're markdown reference files that encode design decisions, platform standards, and craft details.

### Use as Documentation

Read the files in `skills/references/` directly. Each covers one design domain with specific rules, values, and decision frameworks.

### Use with AI Agents

**Claude Code / Codex:** Clone this repo and start a session in the project directory. The `CLAUDE.md` file at the root automatically loads the skills routing table. When you ask your agent to work on design tasks, it reads the relevant reference file.

**Other agents:** Copy the `skills/` folder into your agent's skills directory, or reference the files from your agent's configuration.

**OpenClaw:** Copy `skills/` to `~/clawd/skills/native-mobile-design/` and restart. The skill triggers automatically on design-related tasks.

## How It Was Built

The app was generated by an AI coding agent using only the `AGENTS.md` spec (which encodes rules from the skills). The skills define the standards, `AGENTS.md` translates them into build instructions, and the agent produces code that follows them consistently across every file.
